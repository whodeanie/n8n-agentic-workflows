# Contributing

We welcome contributions. Please follow these guidelines.

1. Fork the repo and create a feature branch.
2. Keep workflow JSON valid. Run `python scripts/validate_workflows.py` before opening a PR.
3. Add tests for new workflows.
4. Update the README if your workflow introduces new patterns.
5. Use semantic commit messages.

## Adding a New Workflow

1. Create a folder `0X-workflow-name/`.
2. Add `workflow.json` and `README.md`.
3. Run validation script.
4. Open a pull request.

## Code of Conduct

Be respectful and constructive in all interactions.
